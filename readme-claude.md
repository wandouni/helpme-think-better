# 电力政策文件思维导图工具 — 需求规格说明书

> 版本：v1.0 | 日期：2026-05-01

---

## 一、产品概述

### 1.1 产品定位

面向电力行业从业者的 **政策文件快速理解工具** 。用户上传电力政策文件（PDF/Word/Markdown），系统调用 DeepSeek API 提炼核心框架与关键条款，生成结构化思维导图，并支持对任意节点/文本发起 AI 解释查询。

### 1.2 核心价值

* **去芜存菁** ：自动过滤空话套话，只保留核心框架和关键条款
* **可视化理解** ：思维导图直观展示政策结构与层级关系
* **按需深挖** ：选中任意内容一键调用 AI 解释，复杂内容自动附带数据/案例辅助理解

### 1.3 技术栈选型

| 层级         | 技术选型                                               | 理由                                                 |
| ------------ | ------------------------------------------------------ | ---------------------------------------------------- |
| 前端         | React + TypeScript                                     | 组件化、类型安全                                     |
| 思维导图渲染 | markmap                                                | 直接消费 Markdown，与 DeepSeek 输出格式天然契合      |
| 后端         | Python FastAPI                                         | 异步高性能，文件解析生态丰富，流式转发 DeepSeek 响应 |
| AI 服务      | DeepSeek API                                           | 用户自行配置 API Key                                 |
| 文件解析     | PyMuPDF (PDF)、python-docx (Word)、原生读取 (Markdown) | 成熟稳定                                             |

## 三、功能模块详细设计

### 3.1 顶部工具栏

#### 3.1.1 上传列表（下拉菜单）

* **触发方式** ：点击「上传列表」按钮展开下拉
* **列表项内容** ：文件名 + 上传时间 + 删除按钮（×）
* **点击行为** ：点击某个文件 → 左侧面板切换到该文件对应的思维导图
* **删除行为** ：点击删除 → 二次确认弹窗 → 确认后从列表移除，同时清除该文件相关的思维导图数据和解释对话记录
* **空状态** ：列表为空时显示「暂无文件，请上传」

#### 3.1.2 上传文件按钮

* **支持格式** ：`.pdf`、`.docx`、`.md`
* **交互流程** ：

1. 点击按钮 → 弹出系统文件选择器
2. 选择文件后 → 顶部工具栏显示上传进度条
3. 上传完成 → 后端解析文件提取文本 → 调用 DeepSeek API 生成思维导图 Markdown
4. 生成完成 → 左侧面板自动渲染思维导图，文件自动加入上传列表

* **加载状态** ：生成过程中左侧面板显示骨架屏/loading 动画 + 文字提示「正在提炼核心内容...」
* **错误处理** ：
* 文件格式不支持 → toast 提示「仅支持 PDF、Word、Markdown 格式」
* 文件过大（建议限制 20MB）→ toast 提示「文件过大，请控制在 20MB 以内」
* 解析失败 → toast 提示具体原因
* DeepSeek API 调用失败 → toast 提示「AI 生成失败，请检查 API 设置」

#### 3.1.3 DeepSeek 设置按钮（⚙图标）

* **触发方式** ：点击弹出模态对话框
* **表单字段** ：

| 字段         | 类型       | 必填 | 说明                                                                  |
| ------------ | ---------- | ---- | --------------------------------------------------------------------- |
| API Key      | 密码输入框 | 是   | 显示为 `••••••`，右侧有显示/隐藏切换                          |
| API Base URL | 文本输入框 | 是   | 默认值 `https://api.deepseek.com`，支持用户修改（兼容自建代理）     |
| 模型名称     | 文本输入框 | 是   | 默认值 `deepseek-chat`，支持用户修改                                |
| 连接测试     | 按钮       | —   | 点击后发送测试请求，成功显示绿色「✓ 连接成功」，失败显示红色错误信息 |

* **持久化** ：配置信息存储在浏览器 localStorage，刷新页面不丢失
* **安全提示** ：对话框底部提示「API Key 仅存储在您的浏览器本地，不会上传到服务器」

#### 3.1.4 解释按钮（💡图标）

* **前置条件** ：必须已配置 DeepSeek API Key，否则点击后 toast 提示「请先配置 DeepSeek API」
* **触发逻辑** （详见 3.4 节）：
* 思维导图模式下：选中节点后点击 → 发起解释查询
* MD 文本模式下：选中文字后点击 → 发起解释查询
* 无选中内容时点击 → toast 提示「请先选中思维导图节点或文本内容」
* **按钮状态** ：
* 默认灰色（无选中内容）
* 有选中内容时变为高亮可点击状态
* 正在请求中时显示 loading 状态，不可重复点击

### 3.2 左侧面板 — 思维导图/文本视图

#### 3.2.1 视图切换

* 面板顶部设置 Tab 切换：「🗺 思维导图」|「📄 Markdown」
* 切换时保留各自的状态（思维导图的缩放位置、文本的滚动位置）
* 默认显示思维导图视图

#### 3.2.2 思维导图视图

 **渲染引擎** ：markmap

* **数据源** ：DeepSeek API 返回的 Markdown 文本（标题层级格式）
* **交互操作** ：
* **缩放** ：鼠标滚轮缩放 / 双指捏合缩放（移动端）
* **平移** ：鼠标拖拽画布平移
* **展开/折叠** ：点击节点的 ⊕/⊖ 图标展开或折叠子节点
* **选中节点** ：单击某个节点 → 节点高亮（边框加粗+背景色变化），同时工具栏「解释」按钮变为可用状态
* **取消选中** ：点击空白区域取消选中
* **工具按钮** （浮动在思维导图右下角）：
* **放大** （+）：放大视图
* **缩小** （-）：缩小视图
* **适应屏幕** （⊡）：自动缩放至完整显示所有节点
* **全屏查看** （⛶）：进入浏览器全屏模式展示思维导图，按 ESC 退出
* **下载 MD** （↓）：将当前思维导图对应的 Markdown 源文本下载为 `.md` 文件，文件名为「原文件名_思维导图.md」

#### 3.2.3 Markdown 文本视图

* **展示内容** ：DeepSeek 返回的思维导图 Markdown 原始文本
* **渲染方式** ：Markdown 渲染展示（带语法高亮的标题层级），非纯文本
* **文本选中** ：用户可用鼠标拖选任意文字片段 → 选中后工具栏「解释」按钮变为可用状态
* **空状态** ：未上传文件时显示「上传文件后，此处将展示思维导图的 Markdown 文本」

### 3.3 右侧面板 — 解释对话区

#### 3.3.1 布局结构

类似即时通讯对话界面，自上而下排列对话记录：

```
┌──────────────────────────┐
│  解释对话                  │
├──────────────────────────┤
│ 🧑 [我的提问内容]          │  ← 右对齐，蓝色气泡
│                          │
│ 🤖 [DeepSeek返回内容]     │  ← 左对齐，灰色气泡
│                          │
│ 🧑 [我的提问内容]          │
│                          │
│ 🤖 [DeepSeek返回内容]     │
│                          │
│       ...                │
├──────────────────────────┤
│  （无输入框，通过选中+      │
│    解释按钮触发）           │
└──────────────────────────┘
```

#### 3.3.2 对话记录

* **我的提问** （右对齐蓝色气泡）：展示实际发送给 DeepSeek 的完整 prompt（包含拼接的「解释一下」或「结合数据或者例子解释，方便记忆和理解」后缀）
* **DeepSeek 回复** （左对齐灰色气泡）：
* 支持 Markdown 渲染（DeepSeek 返回内容可能包含列表、代码块、表格等）
* 流式展示：逐字/逐句出现，体验更好
* **加载状态** ：DeepSeek 回复中时，显示打字机动画（三个跳动的点）
* **错误处理** ：API 调用失败时，在对应回复位置显示红色错误提示 + 「重试」按钮

#### 3.3.3 对话绑定

* 每个上传文件有独立的解释对话记录
* 切换文件时，对话区同步切换到对应文件的对话记录
* 对话记录保持在内存中（页面刷新后丢失，可接受）

#### 3.3.4 空状态

未发起过解释时显示引导文字：「选中思维导图节点或文本内容，点击顶部 💡 解释按钮，即可获取 AI 解释」

### 3.4 解释功能 — 核心交互逻辑

#### 3.4.1 选中内容获取

| 来源              | 选中方式     | 获取选中内容的方法                               |
| ----------------- | ------------ | ------------------------------------------------ |
| 思维导图          | 单击节点     | 获取该节点的文本内容（含子节点文本，按层级拼接） |
| Markdown 文本视图 | 鼠标拖选文字 | 获取 `window.getSelection().toString()`        |

#### 3.4.2 Prompt 拼接规则

根据选中内容的复杂程度自动判断拼接方式：

```
判断逻辑：
  IF 选中内容字数 ≤ 30 字 AND 不含数字/百分比/公式：
    prompt = "{选中内容}\n\n解释一下"
  ELSE：
    prompt = "{选中内容}\n\n结合数据或者例子解释，方便记忆和理解"
```

 **复杂度判定细化** ：

* **简单内容** （拼接「解释一下」）：
* 字数 ≤ 30
* 内容为纯概念/名词/短语
* 例："需求响应"、"辅助服务市场"、"现货市场"
* **复杂内容** （拼接「结合数据或者例子解释，方便记忆和理解」）：
* 字数 > 30
* 或包含数字、百分比、公式
* 或包含多个分号/逗号分隔的并列条款
* 例："按照日前市场出清价格的 110% 设定价格上限"、"第十三条 现货市场按照节点边际电价..."

#### 3.4.3 完整请求流程

```
用户选中内容 → 点击解释按钮
  → 前端按规则拼接 prompt
  → 将完整 prompt 展示在右侧对话区（我的提问气泡）
  → 前端调用后端 /api/explain 接口
  → 后端转发请求至 DeepSeek API（流式）
  → 前端流式接收并逐步渲染在对话区（DeepSeek 回复气泡）
  → 完成
```

## 四、后端 API 设计

### 4.1 接口列表

| 接口                      | 方法 | 说明                                            |
| ------------------------- | ---- | ----------------------------------------------- |
| `/api/upload`           | POST | 上传文件，解析提取文本                          |
| `/api/generate-mindmap` | POST | 调用 DeepSeek 生成思维导图 Markdown（流式返回） |
| `/api/explain`          | POST | 调用 DeepSeek 解释选中内容（流式返回）          |

### 4.2 接口详细设计

#### 4.2.1 POST `/api/upload`

 **请求** ：`multipart/form-data`

| 字段 | 类型 | 说明                      |
| ---- | ---- | ------------------------- |
| file | File | 上传的文件（PDF/DOCX/MD） |

 **响应** ：`application/json`

```json
{
  "file_id": "uuid-string",
  "filename": "某省电力市场规则.pdf",
  "text_content": "提取的纯文本内容...",
  "char_count": 15230
}
```

 **处理逻辑** ：

* PDF → PyMuPDF 提取文本
* DOCX → python-docx 提取文本
* MD → 直接读取原文

#### 4.2.2 POST `/api/generate-mindmap`

 **请求** ：`application/json`

```json
{
  "text_content": "文件提取的文本内容...",
  "api_key": "sk-xxx",
  "api_base_url": "https://api.deepseek.com",
  "model": "deepseek-chat"
}
```

 **响应** ：`text/event-stream`（SSE 流式返回）

```
data: # 政策名称
data:
data: ## 一、总则
data:
data: ### 1.1 目的
data: - 关键条款内容
data:
...
data: [DONE]
```

 **System Prompt（发送给 DeepSeek）** ：

```
你是一个电力政策文件分析专家。请将以下政策文件提炼为思维导图的 Markdown 格式。要求：
1. 只保留核心框架和关键条款，去掉所有空话套话、背景描述、过渡性文字
2. 使用 Markdown 标题层级（# ## ### ####）表示思维导图层级
3. 叶子节点使用无序列表（-）
4. 重点突出：数字指标、具体比例、关键时间节点、处罚条款、核心定义用 **加粗**
5. 保持条款的原始编号（如第X条）作为参考
6. 整体结构不超过 4 级深度
```

#### 4.2.3 POST `/api/explain`

 **请求** ：`application/json`

```json
{
  "prompt": "需求响应\n\n解释一下",
  "api_key": "sk-xxx",
  "api_base_url": "https://api.deepseek.com",
  "model": "deepseek-chat"
}
```

 **响应** ：`text/event-stream`（SSE 流式返回）

```
data: 需求响应是指...
data: ...
data: [DONE]
```

 **System Prompt（发送给 DeepSeek）** ：

```
你是一个电力行业专家，擅长用通俗易懂的语言解释电力政策和技术概念。
请针对用户选中的内容进行解释。要求：
1. 语言简洁清晰，避免重复原文
2. 如果是专业术语，先给出一句话定义，再展开解释
3. 如果用户要求结合数据或例子，请给出具体的数值、场景或类比
4. 控制篇幅，解释在 300 字以内，除非内容确实需要更长的解释
```

### 4.3 安全设计

* API Key 由前端传入后端，后端仅用于转发请求，**不做任何存储**
* 上传文件临时存储在服务器 `/tmp` 目录，解析完成后立即删除
* 接口增加基本的请求频率限制（防止滥用）

## 五、DeepSeek 生成思维导图的 Prompt 工程

### 5.1 Markdown 输出格式规范

DeepSeek 返回的 Markdown 必须符合 markmap 可渲染的格式：

```markdown
# 政策文件名称

## 一、总则
### 1.1 适用范围
- 适用于XX省级及以上电力市场
- 覆盖 **发电侧、售电侧、用户侧**

### 1.2 基本原则
- 市场化定价
- 安全约束下的经济优化

## 二、市场组织
### 2.1 市场成员
- 发电企业：装机 **≥10MW** 的统调机组
- 售电公司：注册资本 **≥2000万**
### 2.2 交易品种
- 日前市场
- 实时市场
```

### 5.2 提炼原则

* **去掉的内容** ：指导思想、总体要求、工作部署、宏观愿景描述、重复性修饰语
* **保留的内容** ：具体数字指标、时间节点、权责划分、计算公式、处罚/奖励条款、核心定义
* **加粗的内容** ：关键数字、比例、金额、时限、新增/变更条款

## 七、状态管理设计

### 7.1 核心状态

```typescript
interface AppState {
  // 文件管理
  files: FileItem[];              // 已上传文件列表
  activeFileId: string | null;    // 当前选中的文件 ID

  // DeepSeek 配置
  deepseekConfig: {
    apiKey: string;
    apiBaseUrl: string;
    model: string;
  };

  // 思维导图
  mindmapMarkdown: Record<string, string>;  // fileId → markdown 内容

  // 视图状态
  leftViewMode: 'mindmap' | 'markdown';     // 左面板当前视图
  selectedContent: string | null;           // 当前选中的文本内容
  selectionSource: 'mindmap' | 'markdown' | null;

  // 解释对话
  chatHistory: Record<string, ChatMessage[]>;  // fileId → 对话记录

  // UI 状态
  isGenerating: boolean;          // 思维导图生成中
  isExplaining: boolean;          // 解释请求中
  splitRatio: number;             // 左右面板分割比例 (0-1)
}

interface FileItem {
  id: string;
  filename: string;
  uploadTime: string;
  textContent: string;
}

interface ChatMessage {
  role: 'user' | 'assistant';
  content: string;
  timestamp: string;
  isError?: boolean;
}
```

## 九、非功能需求

### 9.1 性能

* 文件上传限制 20MB
* 思维导图渲染节点数建议不超过 200 个（超过时 DeepSeek prompt 中增加精炼要求）
* SSE 流式响应保证首字输出延迟 < 3 秒
* 拖拽分隔线帧率 ≥ 30fps，无卡顿

### 9.2 兼容性

* 现代浏览器：Chrome 90+、Firefox 88+、Edge 90+、Safari 15+
* 响应式：最小支持宽度 1024px（非移动端优先产品）

### 9.3 错误处理汇总

| 场景                           | 提示方式                    | 提示内容                               |
| ------------------------------ | --------------------------- | -------------------------------------- |
| 未配置 API Key 时点击上传/解释 | Toast 警告                  | 请先配置 DeepSeek API                  |
| 文件格式不支持                 | Toast 错误                  | 仅支持 PDF、Word、Markdown 格式        |
| 文件过大                       | Toast 错误                  | 文件过大，请控制在 20MB 以内           |
| 文件解析失败                   | Toast 错误                  | 文件解析失败：{具体原因}               |
| DeepSeek API 调用失败          | Toast 错误 + 对话区红色提示 | AI 服务调用失败：{错误信息} + 重试按钮 |
| 无选中内容时点击解释           | Toast 提示                  | 请先选中思维导图节点或文本内容         |
| 网络断开                       | Toast 错误                  | 网络连接失败，请检查网络               |
